# IELTS Reading Reference

这份文件只放参考资料。`SKILL.md` 只保留执行流程。

## 题型速查

| 题型 | 核心能力 | 常见错因 |
|------|---------|---------|
| True/False/Not Given | 逻辑判断 | 混淆 False 和 Not Given |
| Yes/No/Not Given | 观点判断 | 混淆作者观点和事实 |
| Matching Headings | 段落概括 | 被细节干扰，没抓主旨 |
| Matching Information | 信息定位 | 定位到错误段落 |
| Matching Features | 人物/理论匹配 | 张冠李戴 |
| Sentence Completion | 信息提取 | 超字数 / 定位错误 |
| Summary Completion | 信息提取 | 同上 |
| Multiple Choice | 理解 + 排除 | 干扰项没排掉 |
| List of Headings | 段落主旨 | 被首句误导 |
| Table/Flow Chart | 信息提取 | 定位错误 |

## T/F/NG 判断框架

```markdown
**题目陈述：** "{题目原文}"

1. 原文有没有提到这个话题？
   - 没提到 -> NOT GIVEN
   - 提到了 -> 继续

2. 原文和题目是什么关系？
   - 意思一致 -> TRUE
   - 意思相反 -> FALSE
   - 只提到相关话题，但没有题目要求的具体信息 -> NOT GIVEN
```

关键区分：
- FALSE = 原文明确说了相反的事
- NOT GIVEN = 原文没有提供足够信息
- 不能靠常识或逻辑脑补，只能看原文明确表达

### T/F/NG 常见陷阱

| 陷阱 | 示例 | 正确判断 |
|------|------|---------|
| 原文说了 A，题目问 A+B | 原文说 `tea prevented disease`，题目说 `tea and beer prevented disease` | 需要检查 `beer` 是否被提到 |
| 原文用了比较级，题目用了最高级 | 原文说 `more attractive`，题目说 `the most attractive` | 这是不同陈述 |
| 原文说了原因 A，题目说了原因 B | 原文说 `因为X`，题目说 `因为Y` | 如果原文明确排斥 Y，判 FALSE |
| 题目加了限定词 | 原文说 `some`，题目说 `all` | FALSE |
| 原文没提时间/地点 | 题目加了 `in the 18th century` | 检查原文是否提到时间地点 |

## Matching Headings 框架

```markdown
### 段落 {X} 标题匹配

**段落核心：** {一句话概括}
**首句：** "{首句}"
**尾句：** "{尾句}"
**关键词：** {反复出现的主题词}

**正确标题：** {x}
**匹配逻辑：** 标题关键词对应段落里的哪个表述

**干扰标题：** {y}
**排除原因：** 它只是细节，不是本段主旨
```

### Matching Headings 通用策略

- 先读所有标题，划出关键词
- 从最容易确定的段落开始
- 段落核心通常看首句和尾句的交集
- 如果首句只是过渡，主旨往往在后面
- 用排除法缩小剩余选项

## 高频同义替换模式

- important -> significant / crucial / vital / essential
- increase -> rise / grow / expand / surge / climb
- decrease -> decline / drop / fall / reduce / diminish
- cause -> lead to / result in / give rise to / trigger
- difficult -> challenging / demanding / arduous
- show -> demonstrate / indicate / reveal / suggest
